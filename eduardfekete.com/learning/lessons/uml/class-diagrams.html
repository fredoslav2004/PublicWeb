Class Diagrams

Class diagrams are the most common UML diagrams used in software engineering. They describe the structure of a system by showing its classes, attributes, operations, and the relationships among objects.

Basic Class

A class block is divided into three sections:

  1. Name
  2. Attributes
  3. Methods

Relationships

  • Association: A general link between two classes.
  • Inheritance: “is-a” relationship.
  • Composition: “part-of” relationship where the part cannot exist without the whole.

Example

UML Diagram

Code Snippet

public class Dog extends Animal {
    public void bark() {
        System.out.println("Woof!");
    }
}
