AVR Comparison & Branching Logic

Reference for ATmega2560. // Datasheet Compliant

The Mechanism

The instructions CP and CPI perform a subtraction internally (Operand1 - Operand2) to update the Status Register (SREG). They do not store the result.

Flags Used:

  • Z (Zero): Set if operands are equal.
  • C (Carry): Used for Unsigned logic.
  • S (Sign): Used for Signed logic (N ⊕ V).

Hardware Constraints

CP
Compare Registers cp Rd, Rr

Valid for r0 - r31.

CPI
Compare Immediate cpi Rd, K

⚠ Valid ONLY for r16 - r31.

Valid Instruction Matrix

Datasheet Page 457

Important: The AVR instruction set does not contain specific instructions for "Greater Than" or "Less or Equal". You must achieve these checks by either using the available instructions below or by swapping your operands.

Unsigned (uint8_t)

Relies on Carry (C) Flag

Check Instruction Flag Logic
A < B BRLO if (C == 1)
A >= B BRSH if (C == 0)
A == B BREQ if (Z == 1)
A != B BRNE if (Z == 0)

Missing "Greater Than"?

There is no "Branch if Higher" (BRHI).
To check if A > B, compare B to A and use BRLO.

Signed (int8_t)

Relies on Sign (S) Flag

Check Instruction Flag Logic
A < B BRLT if (S == 1)
A >= B BRGE if (S == 0)
A == B BREQ if (Z == 1)
A != B BRNE if (Z == 0)

Missing "Greater Than"?

There is no "Branch if Greater Than" (BRGT).
To check if A > B, compare B to A and use BRLT.

Handling Missing Instructions

Why they are missing

The AVR hardware flags only natively support "Less Than" and "Greater or Equal" checks. Logic like "Greater Than" (Strict) requires checking multiple flags (e.g., Zero must be 0 AND Sign must be 0), which does not map to a single opcode.

The Solution: Operand Swap

Instead of complex branching logic, simply swap the order of operands in the CP instruction.

Desired Logic Standard Way (Impossible) Optimized Way (Valid)
if (r16 > r17) cp r16, r17
brgt target		 cp r17, r16 brlt target (Is r17 < r16?)
if (r16 <= r17) cp r16, r17
brle target		 cp r17, r16 brge target (Is r17 >= r16?)
