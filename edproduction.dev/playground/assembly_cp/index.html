AVR Comparison & Branching Logic

Reference for ATmega2560 / AVR Instruction Set. // Strict Architecture Rules

The Mechanism

Both CP and CPI perform a subtraction internally:

Result = Operand1 - Operand2

The result is discarded. Only the Status Register (SREG) flags are updated. The subsequent BRxx instruction inspects these flags to jump or fall through.

Hardware Constraints

CP
Compare Registers cp Rd, Rr

Works on ALL registers (r0 - r31).

Branch Selection Matrix

Select context carefully

Unsigned (uint8_t)

0 to 255

Condition Instruction Mnemonic Flags
A < B BRLO Branch Lower C=1
A <= B BRLS Branch Lower/Same C=1 | Z=1
A >= B BRSH Branch Same/Higher C=0
A > B BRHI Branch Higher C=0 & Z=0

Note: BRLO is an alias for BRCS (Carry Set). BRSH is an alias for BRCC (Carry Clear). Use the aliases for readability.

Signed (int8_t)

-128 to +127

Condition Instruction Mnemonic Flags
A < B BRLT Branch Less Than S=1
A <= B BRLE Branch Less/Equal S=1 | Z=1
A >= B BRGE Branch Greater/Equal S=0
A > B BRGT Branch Greater Than S=0 & Z=0

Note: The S (Signed) flag is N XOR V. It handles overflow logic automatically. Never use Carry logic for signed math.

Multi-Byte Comparison (16-bit / 32-bit)

To compare 16-bit numbers (e.g., r25:r24 vs r19:r18), you must chain instructions.

  • Step 1: Compare low bytes (LSB) using CP.
  • Step 2: Compare high bytes (MSB) using CPC (Compare with Carry).
  • Step 3: Branch based on the final result.
16-bit Compare Example

; Compare r25:r24 with r19:r18
cp   r24, r18    ; Compare LSB
cpc  r25, r19    ; Compare MSB + Previous Carry

; Now branch based on the FULL 16-bit result
brlo target_label ; Jump if (r25:r24) < (r19:r18)

Equality Checks

Equality logic is identical for both Signed and Unsigned types. It relies solely on the Zero (Z) flag.

BREQ Branch if Equal (Z=1)
BRNE Branch if Not Equal (Z=0)
