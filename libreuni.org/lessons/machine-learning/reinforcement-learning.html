The Agent-Environment Loop

Reinforcement Learning (RL) is a paradigm of machine learning focused on sequential decision making. Unlike supervised learning, where the model receives explicit “labels”, an RL Agent learns by interacting with an Environment. It performs Actions, transitions between States, and receives scalar Rewards.

The goal is to learn a mapping from states to actions—a Policy π(s)—that maximizes the total expected future reward.

1. Markov Decision Processes (MDP)

The mathematical framework for RL is the MDP, defined by the tuple (S,A,P,R,γ):

  • S (State Space): The set of all possible configurations.
  • A (Action Space): The set of all possible moves.
  • P(s′∣s,a) (Transition Dynamics): The probability that action a in state s leads to state s′.
  • R(s,a,s′) (Reward Function): The immediate feedback signal.
  • γ (Discount Factor): γ∈[0,1). It determines the “horizon” of the agent.

The Markov Property states that the future depends only on the current state, not the historical sequence: P(st+1​∣st​)=P(st+1​∣st​,st−1​,…,s0​)

2. Value Functions and Bellman Equations

To optimize a policy, we must evaluate how “good” a state or an action is.

  • State-Value Function Vπ(s): The expected total reward starting from s following policy π.
  • Action-Value Function Qπ(s,a): The expected total reward starting from s, taking action a, and then following π.

The Bellman Optimality Equation

The fundamental recursive relationship in RL: Q∗(s,a)=E[Rt+1​+γmaxa′​Q∗(st+1​,a′)∣st​=s,at​=a] This says the value of the current action is the immediate reward plus the value of the best possible action in the next state.

3. Q-Learning: Model-Free Control

Q-Learning is an Off-Policy algorithm, meaning it can learn the optimal policy even if the agent is behaving randomly. It updates its Q-table using temporal difference error:

Q(s,a)←Q(s,a)+α[r+γmaxa′​Q(s′,a′)−Q(s,a)]

4. Exploration vs. Exploitation

If an agent only executes actions it knows are good (Exploitation), it might never find a superior hidden path. If it only tries new things (Exploration), it will never accumulate high rewards.

Simple Solution: ϵ-greedy Policy

  • With probability ϵ: Take a random action.
  • With probability 1−ϵ: Take the action with the highest Q-value. Common practice is to start with ϵ=1.0 and decay it toward 0.05 as the agent learns.

5. From Tabular RL to Deep RL

When the state space is continuous (e.g., coordinates in physics) or high-dimensional (e.g., raw pixels), a Q-Table is impossible.

Deep Q-Networks (DQN)

Instead of a table, we use a Neural Network Q(s,a;θ) to approximate the values.

  • Experience Replay: Storing past transitions (s,a,r,s′) in a buffer and sampling them randomly to break the correlation between consecutive steps.
  • Fixed Q-Targets: Using a separate, slowly-updating network to calculate the targets, improving training stability.

Policy Gradient Methods

Instead of estimating values, these models directly learn the probability distribution π(a∣s).

  • Advantage: They can handle continuous action spaces (e.g., “apply 12.5 Newtons of force”).
  • Popular Algorithms: PPO (Proximal Policy Optimization), used for training ChatGPT (RLHF).

6. Real-World Applications

  • Game AI: AlphaGo (Chess/Go) and AlphaStar (StarCraft II).
  • Robotics: Learning walking gaits or grasping complex objects.
  • Finance: Portfolio optimization and high-frequency trading.
  • Recommender Systems: Long-term user engagement optimization.

Summary: When to Use RL?

Reinforcement Learning is suitable when:

  1. You have an Environment (simulation or real world).
  2. You can define a clear Reward signal.
  3. The problem involves a Sequence of decisions where early actions affect future possibilities.
  4. Optimal “labels” for specific states are unknown or too expensive to generate.
